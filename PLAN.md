# StrataNodex — Backend Plan
> Node.js + Express.js + TypeScript + Prisma + PostgreSQL (Neon)

---

## What Is This?

StrataNodex is a CLI-first, cross-platform productivity and task management system. This repo is **only the backend** — a REST API that all clients (CLI, Web, Mobile) talk to.

The backend is the single source of truth. No client touches the database directly. Everything goes through this API.

---

## How The App Works (Big Picture)

```
User
 └── Folders           (e.g. "Work", "Personal")
      └── Lists         (e.g. "Project A", "Gym Routine")
           └── Nodes    (Tasks — infinite nesting via parent_id)
                └── Sub-nodes
                     └── Sub-sub-nodes (no limit)
```

Each Node has:
- Title, status (todo / in_progress / done)
- Start & end datetime (for Daily Task auto-population)
- Reminder datetime
- Notes (markdown text)
- Tags (many-to-many, scoped per user)
- Priority (low / medium / high)
- Canvas position (x, y) — used by web app, ignored by CLI
- Parent reference (self-referential for nesting)

**Daily Tasks** are a virtual/dynamic view — not stored separately. Any node whose date range includes today is automatically surfaced as a "today task."

**Gamification** — at end of each day, a score is computed based on completion % and stored in `daily_scores`. This powers the streak system and graphs.

```
≥90% done  → +3 points
60–89%     → +2 points
30–59%     → +1 point
1–29%      →  0 points
0%         → -1 point
```

## Database Schema (Prisma)

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id           String       @id @default(cuid())
  email        String       @unique
  passwordHash String
  name         String?
  dayStartTime String       @default("00:00") // HH:MM format
  dayEndTime   String       @default("23:59")
  createdAt    DateTime     @default(now())

  folders      Folder[]
  tags         Tag[]
  dailyScores  DailyScore[]
}

model Folder {
  id        String   @id @default(cuid())
  name      String
  userId    String
  position  Int      @default(0)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  lists     List[]
}

model List {
  id        String   @id @default(cuid())
  name      String
  folderId  String
  position  Int      @default(0)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  folder    Folder   @relation(fields: [folderId], references: [id], onDelete: Cascade)
  nodes     Node[]
}

model Node {
  id         String     @id @default(cuid())
  title      String
  status     NodeStatus @default(TODO)
  priority   Priority   @default(MEDIUM)
  notes      String?
  startAt    DateTime?
  endAt      DateTime?
  reminderAt DateTime?
  canvasX    Float?     // web canvas position, null = not set
  canvasY    Float?
  position   Int        @default(0) // sibling ordering

  listId     String
  parentId   String?    // null = root node

  createdAt  DateTime   @default(now())
  updatedAt  DateTime   @updatedAt

  list       List       @relation(fields: [listId], references: [id], onDelete: Cascade)
  parent     Node?      @relation("NodeChildren", fields: [parentId], references: [id])
  children   Node[]     @relation("NodeChildren")
  tags       NodeTag[]
}

model Tag {
  id        String    @id @default(cuid())
  name      String
  color     String    @default("#888888")
  userId    String

  user      User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  nodes     NodeTag[]

  @@unique([userId, name])
}

model NodeTag {
  nodeId String
  tagId  String

  node   Node   @relation(fields: [nodeId], references: [id], onDelete: Cascade)
  tag    Tag    @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([nodeId, tagId])
}

model DailyScore {
  id          String   @id @default(cuid())
  userId      String
  date        String   // YYYY-MM-DD — one row per user per day
  totalTasks  Int
  doneTasks   Int
  points      Int
  createdAt   DateTime @default(now())

  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, date])
}

enum NodeStatus {
  TODO
  IN_PROGRESS
  DONE
}

enum Priority {
  LOW
  MEDIUM
  HIGH
}
```

## API Endpoints

### Auth
```
POST   /api/auth/register     → { name, email, password }
POST   /api/auth/login        → { email, password } → { token }
GET    /api/auth/me           → returns current user (protected)
```

### Folders
```
GET    /api/folders           → list all folders for user
POST   /api/folders           → create folder
PATCH  /api/folders/:id       → update folder name/position
DELETE /api/folders/:id       → delete folder (cascades lists + nodes)
```

### Lists
```
GET    /api/folders/:folderId/lists     → all lists in a folder
POST   /api/folders/:folderId/lists     → create list
PATCH  /api/lists/:id                   → update list
DELETE /api/lists/:id                   → delete list
```

### Nodes
```
GET    /api/lists/:listId/nodes         → full tree for a list
POST   /api/lists/:listId/nodes         → create root node
POST   /api/nodes/:parentId/children    → create sub-node
PATCH  /api/nodes/:id                   → update node (title, status, dates etc)
DELETE /api/nodes/:id                   → delete node + all children
PATCH  /api/nodes/:id/move              → change parentId or position
```

### Tags
```
GET    /api/tags                        → all tags for user
POST   /api/tags                        → create tag
DELETE /api/tags/:id                    → delete tag
POST   /api/nodes/:id/tags/:tagId       → attach tag to node
DELETE /api/nodes/:id/tags/:tagId       → detach tag from node
```

### Daily
```
GET    /api/daily/today                 → nodes where startAt <= today <= endAt
GET    /api/daily/overdue               → nodes where endAt < today and status != DONE
```

### Scores
```
GET    /api/scores                      → full score history for user
GET    /api/scores/streak               → current streak count
POST   /api/scores/compute              → manually trigger daily score compute
```

## Request Lifecycle

```
HTTP Request
    ↓
Express Router         (routes/node.routes.ts)
    ↓
Auth Middleware         (verifies JWT, attaches req.user)
    ↓
Validate Middleware     (Zod schema check on req.body)
    ↓
Controller             (calls service, returns res.json)
    ↓
Service                (all business logic, talks to Prisma)
    ↓
Prisma Client          (query to PostgreSQL on Neon)
    ↓
Response
```

## Key Rules (Do Not Break These)

1. **Controllers are dumb** — they only call a service and return `res.json()`. Zero business logic in controllers.
2. **Services own all logic** — every `WHERE userId = ?` check, every calculation, every DB query lives in services.
3. **Never expose passwordHash** — strip it from every user response.
4. **Always scope to req.user.id** — every query must filter by the authenticated user. Never trust IDs from the request body for ownership checks.
5. **Cascade deletes are handled by Prisma** — deleting a folder deletes its lists, which delete their nodes. Never manually clean up children.
6. **Canvas x,y are optional** — CLI never sends them. Backend must treat them as nullable always.
7. **Daily scores are immutable** — once written for a date, never update them. Append-only log.

## How To Run Locally

```bash
# 1. Clone and install
git clone https://github.com/yourusername/stratanodex-backend
cd stratanodex-backend
npm install

# 2. Setup env
cp .env.example .env
# Fill in DATABASE_URL from Neon dashboard and set JWT_SECRET

# 3. Run migrations
npx prisma migrate dev --name init

# 4. Start dev server
npm run dev
```
